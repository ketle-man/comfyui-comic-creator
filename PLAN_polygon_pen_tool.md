## 背景・目的

`eagle_comic_creater_spa`（ComfyUIカスタムノード `comfyui-comic-creater`）の既存機能を拡張する。

1. **テンプレート作成のSPA内蔵化**: 現状「SVGからテンプレート作成」は外部ツール（Illustrator/Inkscape等）でSVGを作成し、アップロードする形式。これをSPA内で完結させたい。
2. **レイアウトタブのドロー機能強化**: 自由なシェイプ（多角形）の作成に対応したい。

両者は同じ「多角形ペンツール」を共通部品として実装し、出力先だけを分ける設計とする。

---

## 現状把握（調査済み事実）

### テンプレート作成（`parseSVGForTemplate`, main.js:3053）

- SVGの規約: `<polygon id="panel_0">` がページ全体のベース（オーバーレイのクリップ範囲、コマには含まれない）、`id="panel_1"` 以降が実際のコマ
- `points` 属性は自由多角形に対応済み（矩形限定ではない）
- 保存先は **IndexedDB**（`dbPut('pages', ...)` 等）。バックエンド（`py/ccc.py`）にテンプレート専用APIは存在せず、完全にフロントエンド完結
- 期待されるデータ構造:
  ```js
  { id: filename, name: filename, width, height, panels: [{ id, number, points }], basePanelPoints }
  ```

### レイアウトタブのドロー機能（`_layerDrawMouseDown/Move/Up`, main.js:11811〜）

- 現在の「曲線」は**1回のドラッグ中に連続して座標を記録するフリーハンド線**（ペンでなぞる方式）
- クリックで頂点を1つずつ追加する「多角形エディタ」は未実装
- 確定した図形は `<path>` / `<rect>` 等としてSVGに追加され、`draw-shape` クラス＋ `data-x/y/w/h` 等の属性でバウンディングボックスを保持し、既存のレイヤー移動・複製・undoシステムに乗る
- 座標変換関数 `_layerDrawClientToSvg` / `_layerDrawSvgToCanvas` は再利用可能

## 設計方針

「多角形ペンツール（`PolygonPenTool`）」を独立した共通コンポーネントとして実装し、以下2箇所から呼び出す。

- **出力先A（今回優先）**: テンプレートタブ → `panel_N` ポリゴン群（`parseSVGForTemplate` 互換のデータ構造）
- **出力先B（後続フェーズ）**: レイアウトタブのドロー → `draw-shape` 要素（既存レイヤーシステム互換）

## フェーズ0: 共通ペンツールの仕様確定 → ❌ 未実施（方式変更のため不要に）

ナイフ（ライン分割）方式を採用したため、頂点クリック式ペンツールの仕様検討は行っていない。
以下は当初案として記録を残す。

現行の「曲線」はドラッグ中の連続点記録（フリーハンド）だが、多角形ペンツールは**クリックで頂点を1つずつ確定**する方式にする。

| 項目 | 内容 | 状態 |
|---|---|---|
| 頂点追加 | クリックで1点確定、次のクリックまでプレビュー線を表示 | 方針決定済み |
| パスを閉じる | 始点付近をクリック／ダブルクリック／Enterキーのいずれか | **要選定** |
| 直前の頂点取消 | Backspace/Escで1点戻せると作業効率が良い | **要検討**（v1に含めるか） |
| 頂点編集 | 確定後にドラッグで頂点位置を修正できるか | **要検討**（v1では省略も可） |
| 出力形式 | SVG座標系での `points` 配列（`x1,y1 x2,y2 ...` 形式、polygonのpoints属性と同一） | 方針決定済み |
